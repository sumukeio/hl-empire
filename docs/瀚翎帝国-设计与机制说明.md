## 1. 产品定位

- **形态**：单页仪表盘（SPA），技术栈为 Next.js 14 App Router + React。
- **叙事包装**：将「个人 / 业务上的最小可行性动作（MVA）」映射为**古风帝国经营**——皇帝属性对应个人状态大盘，城池对应业务单元或产品，军机处对应分时任务，邸报对应操作与系统反馈。
- **数据归属**：全部为**本地优先**；核心状态经 Zustand `persist` 写入浏览器 `localStorage`，无服务端账号体系（除非后续自行扩展）。

| 层级 | 选型 |
|------|------|

## 3. 状态管理与持久化

### 3.1 设计原则

- **按域拆 Store**：皇帝 / 地图 / 任务 / 事件四类独立持久化，互不覆盖 `localStorage` 键名，便于调试与迁移。
- **合并策略（merge）**：皇帝档按 `exp` 重算 `level`；**地图 / 任务**以持久化数组为权威；旧档缺字段时由 `migrateCity` / `migrateQuest` 补默认值（如城池 `troops` / `equipments`）。疆域与军机初始列表由运行时逻辑注入（见 §3.3、§3.4），用户仍可在造办处清空或自定义。

### 3.2 `useEmperorStore`（皇帝 / 系统大盘）

| 字段 | 含义 | 约定 |
|------|------|------|
| `level` | 等级 | 由 `exp` 推导：`floor(exp/100)+1`，下限 1 |
| `exp` | 累计功勋（经验） | 以 `addExp` 等入口正向增加为主 |
| `stamina` | 体力 | 0–100；`consumeStamina` 扣减下限为 0 |
| `gold` | 国库银两 | 业务上可映射收入/现金等 |
| `troops` | 皇帝维度「帝国军力」摘要 | 与**单城池** `City.troops` 不同名同源隐喻 |
| `health` | 龙体健康 | 0–100，默认 100；`setHealth` / 安寝 / 内务府动作等会钳制 |
| `martialArts` | 武术 / 执行力 | 默认 **10**，`Math.max(0, …)` |
| `literature` | 文学修养（理藩院人文线） | 默认 **10**，`Math.max(0, …)` |
| `morale` | 民心 | 0–100，默认 100 |
| `privateVault` | 内帑 / 皇室私库（旅行基金等） | ≥0，默认 0 |
| `healthCombo` | 连续抗炎「连击」计数 | 御膳房 +1，毒丹清零；用于延寿丹邸报阈值 |
| `isNomadMode` | 移动行宫（游牧办公） | 默认 `false`；开启时军机 **仅 `quest-mva-*` 模板** 功勋结算 **×1.2**（见 `quest-store`） |
| `tokens` | 翻牌券 | 主要由军机功勋注入 **`dopaminePool`** 后按 **每满 15 点功勋铸 1 券**（`feedDopamineFromQuestReward`）；御花园 **`startEntertainment`** 每次消耗 **1** 券 |
| `dopaminePool` | 多巴胺蓄池余量 | **0–14**，与功勋结算同 persist；旧档缺省合并为 **0** |
| `isDressed` / `isEntertaining` / `entertainmentDeadline` / `entertainmentLowHealthPenalty` | 朝服与御花园娱乐态 | 见内务府奏折区 |

**内务府 / 养正司相关动作**（实现于 `emperor-store.ts`，多数会经 `useEventStore.getState().addLog` 写邸报）：

- **太医院**：`consumeAntiInflammatory`（体力+10、健康+1、连击；连击≥3 金色延寿邸报）、`trainMartialArts`（体力−5、武力+2）、`consumeJunkFood`（体力/健康惩罚、连击清零）。
- **户部**：`recordExpense(amount, 'infrastructure' \| 'decadence')`（基建扣实额；骄奢双倍扣金 + 民心−10）、`transferToPrivateVault`（国库→私库）。
- **理藩院**：`visitImperialArchives`（私库≥500）、`indulgeCommercialTourism`（私库≥1000）、`setNomadMode`。
- **宗人府**：`rejectInvalidSocial`、`meetGrandSecretariat`（需体力≥5）、`acceptEntropyCompanion`（私库腰斩、民心大损）。
- **户部司帑（顶栏国库 Dialog，亦属户部/内帑叙事）**：**`injectGold(amount)`** — 登记工资/补贴等岁入，`gold` 增加并邸报 `treasury`：「【内务府】岁入拨入 … 两。」；**`syncGold(total)`** — 将 `gold` 校准为实存整数并邸报：「【户部】国库储蓄已校准…」。另保留 **`updateGold`** 供内部与其它模块直接改写（工部拨款、奏折 CPA 联动等）。
- 另有：`sleepAtYangxinPalace`、`jungleHunt`、`royalSparring`、`startEntertainment`、`completeEntertainmentSession` 等宫务向动作。

**其它**：`addExp` 仅叠加功勋与尊号晋升检测（**不**写入多巴胺池）；**`feedDopamineFromQuestReward`** 专供军机点卯链路。**`dailyReset()`** 仍可将体力置满（宵禁整顿等）；跨日军机重置时见 **`resetDailyQuests`**（§3.4）同步清空各城 **`completedQuestIds` / `questCompletedAt` / `questDailyCompletions`**。

**持久化键名**：`hanling-emperor`；`partialize` 含 **`tokens`、`dopaminePool`** 等上述业务字段；`merge` 对缺省档补 `martialArts`/`literature` 默认 **10**、**`dopaminePool` 缺省为 0** 等。

### 3.3 `useMapStore`（疆域 / 业务产品大盘）

**城池结构**（`store/types.ts`）：

| 字段 | 说明 |
|------|------|
| `id` | 稳定唯一标识（新增时由 `createEmptyCity` 等生成） |
| `name` | 叙事名（如「长安」） |
| `alias` | 别名，对应实际产品 / 业务名（如「档案管理员」）；沙盘卡片在名称下展示，可为空 |
| `status` | `0 \| 1 \| 2 \| 3`，见下表 |
| `memo` | 备忘 |
| `cpa` / `orders` | 数值型业务指标（度支 / 粮饷单） |
| `troops` | **单城**帝国兵力：对应后台关键词数量等业务计数 |
| `equipments` | **单城**军械装备：创意图 / 高级样式等计数 |
| `completedQuestIds` | 当日对该任务至少勘合过一次的模板 `id` 列表（与 **`questDailyCompletions`** 中 count>0 同步，兼容旧档仅列表） |
| `questCompletedAt` | 各任务 id → **最近一次**勘合时间戳（ms）；多次勘合同日会更新为末次 |
| `questDailyCompletions` | 当前自然日内各任务 id → 已勘合次数（1–99）；跨日 **`resetDailyQuests`** 整表清空 |

**城池政务与军机联动**：军机处须先选 **主攻城池**（`useQuestStore.activeCityId`）。`toggleQuest(questId)` 在未满 **`maxCompletionsPerDay`** 且体力足够时：**递增**当前城的当日次数、维护 `questCompletedAt`、扣体力、`addExp`（游牧下 **`quest-mva-*`** 功勋 **×1.2** 取整）、再 **`feedDopamineFromQuestReward(expGain)`** 写入多巴胺池并可能铸 **`tokens`**；成功返回 **`ToggleQuestCompletionMeta`**（含铸券数、结算后池值等，供邸报 `revert`）。`toggleQuest(questId, { clearDay: true })` 清空该城该任务**当日**次数（不回退体力/功勋/池与券，与既有约定一致）。`map-store` 内 **`incrementQuestCompletion` / `clearQuestCompletionsForDay` / `decrementQuestCompletion`** 维护计数与 `completedQuestIds`；邸报撤回勘合调用 **`decrementQuestCompletion`**。只读辅助：**`getQuestDailyCount`**、**`isQuestFullyCompletedToday`**（从 `@/store` 或 `map-store` 导出）。

**动作**（补充）：无 `toggleCityQuest`；勘合态以当日计数与阈值为准。

**`status` 语义与 UI 色带**（由 `lib/city-status.ts` 驱动）：`0` 未知之地、`1` 斥候侦查、`2` 战火纷飞、`3` 金色藩属。其中 **`status === 3`** 的沙盘卡片额外使用强金色阴影 + `tailwind` 自定义 **`animate-imperial-gold-glow`**（见 `tailwind.config.ts`），强化「藩属」成就感。

**军备与战力提示**（`lib/city-military.ts` → `isCityEquipmentCritical`）：当 `troops > 0` 且 `equipments === 0`，或 `troops / equipments > 100` 时，中央沙盘城池卡片右上角显示 **红色 `AlertTriangle`**，`Tooltip` 文案：「军备严重不足，战力腰斩，请速派工部打造军械！」

**动作**：

- `addCity(city)`、`removeCity(id)`：增删城池，**无数量上限**。
- `updateCity(id, data)`：`data` 为 `CityPatch`（不可改 `id`），用于朱批侧栏与造办处表单。
- **`bulkAddCities(names: string[])`**：接收名称数组；按名称去重、跳过与现有城池同名项；新城池默认 `status=0`、`cpa=0`、`orders=0`、`troops=0`、`equipments=0` 等；**追加在列表末尾**；返回实际新增数量。名称解析见 **`lib/parse-bulk-city-names.ts`**（支持中文逗号、顿号、英文逗号、空白分隔）。

**持久化键名**：`hanling-map`。无有效存档或首次解析失败时，`merge` 返回 **空疆域 `[]`**。可选用导出方法 `createDefaultCities()` 批量生成示例城列表（含新字段默认值）。

### 3.4 `useQuestStore`（军机处 / 分时任务）

**结构**：

- `quests`：任务**模板**数组；每条含 `period`、`title`、`completed`（遗留字段，勘合态以**城池当日计数**为准）、`expReward`、`staminaCost`、**`sortOrder`**（同时辰内排序）、**`maxCompletionsPerDay`**（本城本自然日该模板最多勘合次数，默认 **1**，造办处可调 1–99）；`id` 稳定。
- `lastLoginDate`：本地日历日 `YYYY-MM-DD`，跨日重置用。
- **`activeCityId`**：`string | null`，军机处 **主攻城池**；未选择时不可勘合。

**默认模板「祖宗之法」（MVA 十四务）**  

`createDefaultQuests()`（`store/quest-store.ts`）提供与业务 MVA 对齐的 **14 条固定任务**，按时辰分布为：**早朝 4、晌午 3、傍晚 4、深夜 3**。任务 `id` 形如 **`quest-mva-{时辰}-{序号}`**（如 `quest-mva-早朝-1`），与历史上旧版打包 id **`quest-default-*`** 区分。

**动作**：

- `addQuest(quest)`、`removeQuest(id)`、`updateQuest(id, data)`：枢密院（造办处）维护任务模板与数值；**无条数上限**。
- **`toggleQuest(questId, opts?)`**：须 **`activeCityId`** 指向有效城池。若 **`opts?.clearDay === true`**：调用 **`clearQuestCompletionsForDay`**，清空该城该任务当日次数（不扣体力）。否则：已达当日上限或体力不足则返回 **`false`**；否则 **`incrementQuestCompletion`** → `consumeStamina` → **`addExp`**（游牧 + **`quest-mva-*`** 时功勋 **×1.2** 取整）→ **`feedDopamineFromQuestReward`**；成功时返回 **`ToggleQuestCompletionMeta`**（非 `boolean`），军机邸报可带完整 **`revert`**（含 **`tokensSubtracted` 为实际铸券数**、`postDopaminePool`、`dopamineExpFed`）供撤回。旧邸报仅有 **`tokensSubtracted: 1`** 且无池字段时，撤回仍按旧逻辑扣券、不调整 **`dopaminePool`**。
- **`setActiveCityId(id)`**：切换主攻城；任务行展示 **`getQuestDailyCount` / `isQuestFullyCompletedToday`** 与 **点卯按钮** 文案（「点卯 / 再办一次 / 本日已办满」）。
- `resetDailyQuests()`：跨自然日：更新 `lastLoginDate`，任务表 `completed` 清零，并 **`useMapStore`** 清空所有城的 **`completedQuestIds` / `questCompletedAt` / `questDailyCompletions`**（邸报 **`logs`** 不随重置清空）。
- **`ensureMvaQuestCatalog()`**（启动时由 **`EmpireCloudSync`** 在 `rehydrate` 之后调用，见 §2）：  
  - 若任务列表为空 → 写入完整 `createDefaultQuests()`；  
  - 若仍存在任一条旧版 **`quest-default-*`** → **整表替换**为当前 MVA 十四务（一次性迁移旧存档）；  
  - 若已有部分 **`quest-mva-*`** → 仅 **补全缺失的默认 id**，不删除用户在造办处自建条目。

**持久化键名**：`hanling-quest`。无有效存档或首次解析失败时，`merge` 可能得到空任务；启动逻辑会通过 `ensureMvaQuestCatalog` 补全默认集。

### 3.5 造办处（系统配置 UI）

- **入口**：国库 HUD 右侧 **造办处齿轮**，`Tooltip`「造办处 (系统配置)」，路由 **`/settings`**。
- **实现**：`components/settings/settings-view.tsx`，**Tabs**：「疆域司」「枢密院」「**帝国档案**」；变更即时写入 `persist`（localStorage），CRUD 多伴随邸报。

**帝国档案**：

- **封装帝国密函**：汇总 `emperor`（含 **`dopaminePool`**）/ `map.cities` / `quest`（含 `activeCityId`）/ `events.logs` 为 JSON 下载，文件名 `hanling-empire-backup-YYYYMMDD.json`（`lib/empire-backup.ts`）。
- **读取帝国密函**：解析后覆盖各 Store（迁移仍走 `migrateCity` / `migrateQuest`）。

**疆域司**：

- 单城维护：城池名、产品别名，以及 **CPA、Orders、兵力（`troops`）、军械（`equipments`）** 数字字段（与沙盘侧滑奏折一致）。
- **批量拓土**：疆域司 Tab 顶部 **多行文本框 +「批量拓土」**；解析规则见 `parseBulkCityNamesFromText`；调用 `bulkAddCities`；成功时邸报 **`decree`**：「造办处：圣上威武，帝国版图今日新增 [N] 座城池！」。

**枢密院**：按四时辰分组维护军机模板（与 §3.4 同一 `quests` 数据源）；每条可编辑 **本日勘合上限**（`maxCompletionsPerDay`，1–99）。

### 3.6 `useEventStore`（八百里加急 / 邸报）

| 字段 / 动作 | 说明 |
|-------------|------|
| `logs` | 时间倒序；单条含 `id`、`time`、`message`、`type`，可选 **`cityName`**、**`emphasis`**（`calamity` 国难红闪、`goldFlash` 金色、`crimsonDecree` 朱红捷报块） |
| `addLog(message, type?, meta?)` | `type` 默认 `info`；`meta` 可含 `cityName`、`emphasis`、**`revert`**（**`quest_complete`**：含 `tokensSubtracted`、可选 **`postDopaminePool` / `dopamineExpFed`** 以支持多巴胺池回滚） |
| `clearLogs()` | 清空 |
| **`revertLog(id)`** | 若该条含 **`revert`**：调用 **`lib/apply-event-log-revert`**（先校验 **`dopaminePool`** 与记录一致再 **`decrementQuestCompletion`**）+ **`revertQuestCompletionEffects`**；成功后从列表移除。失败原因含 `state_mismatch`（勘合已变/跨日清空/池不一致）、`city_missing` 等；实现可向 Console 输出 **`[Hanling]`** 前缀便于排查 |

**邸报 UI（`components/dashboard/event-log-docket.tsx`）**：

- 仅当 **`log.revert` 存在** 时显示**金色描边「撤回」按钮**；否则显示 **「— 无可撤回」** 文案（**非**「禁用按钮」：旧版禁用态带 `pointer-events-none`，易被误认为坏交互）。
- **是否显示「撤回」与「隔了多久」无关**；无「几分钟后自动不可撤」的 UI 逻辑。跨日后当日勘合已清空时，**仍可能显示**「撤回」按钮（`revert` 仍在 JSON 中），但点击会 **失败** 并提示 **state_mismatch**。
- 面板标题区展示 **「本列表可撤回：N 条」** 统计与上述说明。

**容量**：持久化最多保留 **80** 条；**`EventLogPanel`**（顶栏卷轴 Dialog）列表最多渲染最近 **80** 条（与持久化上限一致）。

**持久化键名**：`hanling-event`。

## 4. 跨模块联动（当前实现）

| 场景 | 行为 |
|------|------|
| 军机处勘合（主攻城已选） | `toggleQuest` → `map-store` 当日计数递增 + 体力/功勋 + **多巴胺池铸 `tokens`**；邸报可带完整 **`revert`**、`cityName` |

## 5. UI 信息架构

整体由 **`components/dashboard/dashboard-shell.tsx`** 拼装。

1. **顶部吸顶：国库 HUD（`TreasuryHUD`）**  
   - **左侧**：位阶名（muted）+ **尊号**（`imperial-gold` 高亮）+ `Lv` 与累计功勋；其下为距下一尊号所需功勋（巅峰阶为已满阶说明）；**全宽主进度条**映射 **当前位阶内** 功勋占比（非全生涯总长）。实现见 **`lib/emperor-title.ts`** + **`getEmperorTitleProgress`**。  
   - **中部**：`EmpireBriefingStrip` — 疆域座数、军机务数、粮饷单合计、度支 Σ；若 **`isNomadMode`** 则显示 **「⛺ 移动行宫」** 徽章（提示 MVA 功勋加成）。  
   - **右侧**：**国库银两**（可点击打开 **户部司帑** Dialog：岁入登记 / 现金校准）、帝国军力摘要、**八百里加急（邸报）**卷轴、**勤政录（日历）**→ **`/dashboard/activity`**、**造办处**齿轮（`/settings`）。  
   - **不再**在 HUD 展示龙体/体力条（已迁入下方 **内务府** 奏折区）。

2. **主区：中央沙盘（`WarMap`）**  
   - 响应式网格：`2` → `md:4` → `xl:5` 列。  
   - 标题区 **态势多选筛选条**（四类名称 Chip，默认「战火纷飞」；全取消=不筛选）；网格仅渲染可见态势城池。  
   - 卡片：名称、态势、`CPA/单/兵/械`、备忘摘要；军备临界 **AlertTriangle**（§3.3）。  
   - **邸报捷报**：`OracleBriefing` 等（若有）。  
   - **Sheet 奏折**：别名、疆域态势、**建设进度**（四部：`lib/quest-imperial-axis.ts`，以 **`isQuestFullyCompletedToday`** 判定条目标满；下钻展示当日 **n/max** 与末次勘合时间）、`memo`、数值字段与朱批保存。**保存时若 CPA 上调**：从皇帝 **`gold`** 按差额支用（见底为 0），见 §4「沙盘奏折保存」。

3. **右侧栏（大屏 ≥ lg）：军机处 | 养正司 | 内务府**（同一 `aside` 内横向分栏）  
   - **`QuestEngine`**：顶栏 **主攻城池**；**`lg` 及以上** 用 `Select`（**下拉面板顶部**内嵌搜索框，按城名/别名子串过滤，展示「已匹配 N 座」；触发器上方有操作提示）；**小屏** 用 **底栏 Sheet**，内嵌搜索 + 「已匹配」提示。按时辰分组；每务 **「点卯 / 再办一次 / 本日已办满」** 按钮 + **`getQuestDailyCount` / `isQuestFullyCompletedToday`** 文案；游牧时提示 MVA +20%。  
   - **`InnerCourtCard`（养正司）**：`components/inner-court.tsx` — 太医院 / 户部度支 / 理藩院（内帑行 + 游牧开关）/ 宗人府。  
   - **`NeiwufuCabinet`（内务府）**：皇帝圣躬（健康/体力条、危急闪烁 `animate-imperial-vitals-alert`）、武术、朝服/宵禁、御苑翻牌（**翻牌券 + 多巴胺蓄池条 + 券数回弹动画**）、校场等。

4. **小屏（< lg）**  
   - 底栏按钮 **「军机处 · 养正与宫务」** 打开 **Sheet**（`sr-only` 标题「军机处 · 养正司与内务府」）；内为军机 +（养正司叠于内务府上）纵向滚动；主区 `pb` 防遮挡。

5. **全局叠加**  
   - **`ImperialRedLineWatch`**：红线规则与邸报联动（若存在）。  
   - **邸报（八百里加急）**：入口在 **`TreasuryHUD`** 顶栏 **卷轴按钮**，以 **Dialog** 展示 **`EventLogPanel`**（最多 **80** 条）；**撤回**仅对带 **`revert`** 的军机点卯邸报显示金色按钮；成功/失败横幅与 Console **`[Hanling]`** 日志见 §3.6；支持 **`emphasis`**、`cityName`、清空。  
   - **勤政录**：入口在 **`TreasuryHUD`** 顶栏 **日历按钮**，路由 **`/dashboard/activity`**（日/周/月过滤 + Markdown/JSON 导出，见文首「近期实现更新」）。

## 6. 视觉与主题

- **基底**：`app/layout.tsx` 根节点 `className="dark"`，整体为**深色模式**。
- **皇家主题色**（Tailwind 扩展 `imperial`）：  
  - **暗金** `#F59E0B` → `imperial-gold`，用于尊号点缀、国库意象、主操作强调等。  
  - **朱红** `#E11D48` → `imperial-vermilion`，用于军力、战火态势、邸报标题等。
- **shadcn 语义色**：`globals.css` 中 `--primary` 偏金、`--accent` 偏朱红，与组件默认 variant 一致。
- **城池卡片**：背景与边框随 `status` 变化；**藩属（3）** 叠加强金阴影与呼吸流光动画（见 §3.3）。沙盘区块外层可使用 **`TooltipProvider`** 以支持军备警告的悬浮说明。

**军机处按钮态**：

- 未满档：**点卯** 或 **再办一次**；已满档：**本日已办满** 且禁用；数据以 **`questDailyCompletions`** 与 **`maxCompletionsPerDay`** 为准。

**圣躬危急**（`tailwind.config.ts` + `neiwufu-cabinet.tsx`）：健康 **< 30** 或体力 **< 40** 时，龙体/御体条启用 **`animate-imperial-vitals-alert`** 闪烁（与 `lib/imperial-vitals.ts` 宵禁/早朝阈值一致）。

## 8. 已知局限与可扩展方向（非承诺）

- 军机「`clearDay`」不回退体力/功勋/多巴胺池与券；**邸报撤回**可回滚单次勘合（体力、功勋、**按记录扣回 `tokens`**、**`dopaminePool`**（若邸报含池字段）+ **`decrementQuestCompletion`**）。若需其它可逆路径，再扩展对称扣减与校验。
- 邸报为本地环形记忆（**80** 条持久，与 **`EventLogPanel`** 展示上限一致），无多端同步；**全量存档**请用造办处 **帝国档案** JSON 密函（见 §3.5）。**勤政录**可按日/周/月导出当前内存邸报子集（见文首「近期实现更新」）。
- 城池编辑为右侧 **Sheet**；造办处已支持 **批量拓土**；表格化批量改指标可再扩展。
- 尊号十二阶与数值型 `level`（每 100 功勋一级）未强制绑定；若需统一可改为单一派生。
